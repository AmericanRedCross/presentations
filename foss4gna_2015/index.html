On the ground assessment tools are terrible for conducting a rapid assessment. To much data is recorded on paper which is hard to keep track of, prone to bad encoding, and time consuming to analyze.
We are making great strides on our recogmendations from the OSM-Assessment
Field Papers is awesome. However, its not good really structured assessments. Lots of time is spent just encoding the data after its been collected.
World Bank and OpenCities did some work to create structured forms. These still take lots of time to encode and push to OSM. We needed a tool that would do all of this in one phase. Decreasing the time from data collection to decision.
We approached USAID with an idea to build an app to allow us to collect OSM data before and after disasters. They loved the idea and we started work late last year.

We think about ODK survey answers like putting stuff into buckets. Good tagging in OSM is very similar, we don't versions of the same tag. We want the same tag.

OSM as a

Media Type

After some researh we decided to make OSM its own media type in ODK. This would make changes in java rosa much more lightweight and allow us to focus on the app and not the 5 year old code in JavaRosa.
Same layout as traditional xlsforms but we added the osm tab. We made changes to pyxform to handle the new osm media type to take the xlsform to the formhub api.
Same layout and workflow at ODK. Key thing to note is the use of the osm question type.
